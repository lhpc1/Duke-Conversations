END POINTS

  • POST "/login" with valid username and password credentials will generate a new user session
  • GET "/logout" will logout the current user
  • GET "/user/[int:id]" will return a user's information by id
  • GET "/users/" will return information on all users in the db
  • GET "/userinfo" will return information on the currently logged in user
  • GET "/professors" will return a list of all professors
  • GET "/dinners" will return a list of all dinners
  • GET "/students" will return a list of all students
  • GET "/professor/[string:uniqueID]" will return a specific professor by Unique ID
  • GET "/student/[string:netID]" will return a specific student by netID
  • GET "/dinner/[int:id]" will return a specific dinner by id
  • GET "/dinner/selective?id=[userID]&status=[status]" will return all dinners with a specific status code. 0 indicates unclaimed, 1 is claimed, 2 is finished. User ID refers to all the dinner's of a specific user
  • GET "/dinner/confirm/[int:id]" will set a dinner's status to complete and will EMAIL ALL ACCEPTED APPLICATIONS
  • POST "/application/register" will add a new application to the database
    • "interest": String
    • "studentID": String (FK)
    • "dinnerID": String (FK)
  • POST "/application/update" will bulk update many application statuses simultaneously

    • This endpoint accepts a dict where keywords (Strings) are the ids of the applications and entries (ints) are their new statuses

  • POST "/application/checkin" will bulk update many applicatio, marking them as present

    • This application accepts an array of applicaiton numbers given under the keyword "present"

  • POST "/professor/register" will create a new professor object in memory
    • "uniqueID": String
    • "firstName": String
    • "lastName": String
    • "email": String
    • "genderPronouns": Integer
    • "department": Integer
    • "title": String
    • "school": Integer
  • POST "/dinner/register" will create a new professor object in memory
    • "timeStamp": String
    • "topic": String
    • "description": String
    • "studentLimit": Int
    • "address": String
    • "dietaryRestrictions": String

  • POST "/student/register" will create a new professor object in memory
